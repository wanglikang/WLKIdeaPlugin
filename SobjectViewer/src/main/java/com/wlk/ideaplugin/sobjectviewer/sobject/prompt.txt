如何借助idea插件，实现一个类似数据库表结构查询的窗口插件，要求：

1、在idea的侧边工具栏内，可以打开一个树状的窗口，窗口内一级对象，是各个不同对象的展示，二级对象，是每个对象的所有字段的展示
2、在树状窗口内，双击某个对象，会在编辑器区域打开一个新的tab页，展示这个对象的所有字段的信息
3、在新的tab页内，展示的内容，是一个结构化的对象，对象的每个属性，都是这个对象的一个字段，属性的名称，是字段的名称，属性的值，是字段的类型，字段的类型，是一个枚举类，枚举类的每个值，都是一个字段的类型
4、在树状窗口的顶部，新增几个组件
+ 新增一个查询按钮，用户点击后，会调用某个命令行工具，这个命令行工具也是可以配置的
+ 新增一个下拉框，下拉框的内容，是通过调用一个命令行工具得到的结果，，用户可以从下拉框中选择一个对象，然后点击查询按钮，会查询附带此下拉框参数的的所有字段的信息

5、将查询按钮的逻辑抽离出来，封装为一个方法
6、代码实现，要求高内聚、低耦合，每个类的职责单一，类之间的调用关系清晰，扩展性强

针对上面的功能要求，先给出完整的思路（有哪些类需要新建，各个类之间的调用关系），然后在进行代码生成，生成的代码路径在 com.wlk.ideaplugin.soqlsupport.sobject下
